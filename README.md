# Tour Agency Management System

## Overview
This project is a Laravel application for tour agencies.  
You use it to manage tours, bookings, clients, and payments in one system.  
The focus is speed, clarity, and daily operational control.

## Project Status

> **Note:** This project is currently in development mode.  

## Purpose
You run tours.  
You manage customers.  
You follow reservations and payments from one dashboard.

## Features
- Tour management  
  Create and update tours with prices and schedules.

- Booking system  
  Register reservations and track their status.

- Customer management  
  Store client profiles and booking history.

- Payment tracking  
  Monitor payments, balances, and confirmations.

- Admin dashboard  
  View activity, sales, and key metrics.

- Role access control  
  Define permissions for admins and staff.

- Responsive interface  
  Works on desktop and mobile devices.

## Tech Stack
- Backend  
  Laravel

- Database  
  MySQL or compatible engine

- Frontend  
  Blade templates with a clean UI

- Authentication  
  Laravel authentication system

- Payments  
  Stripe or local payment gateway

## Project Structure
- app  
  Core business logic

- routes  
  Web and API routes

- resources  
  Views and frontend assets

- database  
  Migrations and seeders

- public  
  Entry point and static files

## Installation
1. Clone the repository  
2. Install dependencies using Composer  
3. Configure environment variables in the .env file  
4. Run database migrations  
5. Start the local development server  

## Usage
- Log in as an admin  
- Create tours and schedules  
- Manage bookings  
- Track customers and payments  

## Target Users
- Tour agencies  
- Travel operators  
- Local guides with structured tour services  

## Goals
- Reduce manual booking work  
- Improve customer follow-up  
- Increase operational visibility  
- Support business growth with structured data
